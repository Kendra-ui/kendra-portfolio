Carte Jeune Biométrique — MINJEC (ONJ)

ITGSTORE · Développeuse full stack & analyste applicatif · Août 2024 — aujourd'hui

preenrolement.servicescartejeuneonj.cm ↗

Refonte de la plateforme nationale de pré-enrôlement des jeunes pour l'Observatoire National de la Jeunesse : une application desktop de saisie biométrique pour les agents de terrain, adossée à un back-end Laravel/Spring Boot hébergé sur Oracle Cloud, et un serveur d'enrôlement public en Java/Tomcat.

Aperçu de la plateforme

Application desktop (Java)Lecteur biométrique SecuGen Laravel / PHPPostgreSQL 16Apache2 Interface responsive Oracle Cloud InfrastructureSupervisor (queues) API RESTDiagnostic JVM / GC tuning Certbot / Let's EncryptKeystore PKCS12

Ma contribution

  • Développement du site web de pré-enrôlement (Laravel/Blade), avec une interface responsive qui s'adapte à la taille de l'écran (media queries CSS) : ordinateur, tablette et mobile.
  • Développement de l'application desktop « poste agent » : un exécutable dédié installé sur les postes des agents de l'ONJ pour la recherche de candidats, la capture d'empreinte digitale via lecteur SecuGen, la prise de photo, l'import recto/verso de la CNI, puis la finalisation de l'enrôlement via API REST.
  • Rédaction des guides utilisateur et d'installation destinés aux agents de terrain (utilisation du poste agent, mise en service du lecteur d'empreintes).
  • Déploiement et configuration du serveur de pré-enrôlement Laravel sur Oracle Cloud : Apache2, PostgreSQL 16, file d'attente gérée par Supervisor, tâches planifiées via le scheduler Laravel.
  • Intervention critique sur le serveur d'enrôlement public : diagnostic d'une saturation de la heap JVM Tomcat (« GC death spiral ») ayant rendu le site inaccessible malgré un service techniquement actif ; capture d'un heap dump, reconfiguration de la JVM (heap porté de ~930 Mo à 4 Go, bascule sur le ramasse-miettes G1GC) et rétablissement du service.
  • Remise en service du site d'enrôlement lors d'un second incident : réactivation du service systemd Tomcat, correction des règles iptables Oracle bloquant les ports 80/443, résolution d'un conflit avec une instance Tomcat fantôme sur le port 8005.
  • Diagnostic d'une désynchronisation entre le certificat SSL renouvelé par Certbot et le keystore PKCS12 lu par Tomcat (terminaison TLS assurée directement par Tomcat, sans reverse proxy) : régénération du keystore, redémarrage maîtrisé du service, puis mise en place d'un deploy-hook Certbot pour automatiser le renouvellement de bout en bout.
